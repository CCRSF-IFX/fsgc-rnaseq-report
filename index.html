Study summary

How to use this report

Start with the embedded pipeline results, then add sample metadata or a replacement count matrix when you need browser-side DESeq2 and fgsea.

Recommended workflow

  1. Review the runUse Overview and QC to confirm sample count, contrasts, warnings, genome, annotation, and pipeline provenance.
  2. Check the data setupUse Data Setup to confirm count/sample matching. Upload a sample manifest for the embedded count matrix, or upload a new count matrix with its matching manifest.
  3. Explore sample structureUse PCA, Heatmap & Clustering, and Gene Counts to inspect grouping, batch effects, outliers, and marker-gene behavior.
  4. Run browser DESeq2 when neededChoose one primary factor, set reference/numerator/denominator levels, and optionally adjust or block by batch, sex, subject, or pairing columns.
  5. Run fgsea from a DE resultSelect the DE contrast, upload one or more GMT pathway files, set pathway size limits, and run fgsea in the browser.
  6. Export or reload resultsUse table CSV exports for individual result tables and Methods & Export to save or reload the DESeq2/fgsea analysis cache.

Input files

Count matrix CSV

Required for a build. Include gene_id, gene_symbol, or gene_name, plus one numeric count column per sample.

Sample manifest

Required for user-driven DESeq2 and fgsea. Include sample_id values that match count matrix columns and at least one grouping column with two or more levels.

Optional pipeline outputs

PCA, sample distance, DE, enrichment, QC, provenance, and software-version files are displayed when present. Missing PCA and distances can be computed from counts.

GMT pathway files

Required for browser fgsea. Use pathway files that match the study species and the gene identifiers in the DE table.

GMT files for GSEA

Choose GMT files by species, collection, and gene identifier before uploading them in the GSEA tab.

Official resources Human MSigDB Mouse MSigDB GMT format

Where to download

Open the matching MSigDB collection page, choose a collection row, then use Download GMT Files. A free MSigDB sign-in may be required for downloads.

What to start with

Start with Hallmark for a broad first pass. Use curated pathway collections such as C2/CP or M2/CP for pathway-focused review, and GO collections such as C5 or M5 for ontology terms.

Identifier choice

Select Gene Symbols when the DE table has gene_symbol. Select NCBI/Entrez IDs only when the ranked gene names are Entrez IDs. Do not mix human and mouse GMT files.

Upload in this report

After DESeq2 finishes, go to GSEA, select one or more .gmt files, and run fgsea. Keep downloaded GMT files with the report because analysis caches do not embed them.

Analysis notes

PCA

Color by one metadata factor and, when available, shape by a second factor. Use 3D projection when at least three PCs are available.

Expression heatmap

Show top variable genes or choose a custom gene list. The custom list accepts gene symbols, Ensembl IDs, or gene names separated by commas, spaces, semicolons, tabs, or new lines.

DESeq2

The browser runner supports one primary factor with optional additive covariates or blocking factors. It checks for at least two samples per group and guards against over-specified designs.

Runtime packages

DESeq2 and fgsea use webR. Install/load packages from the configured snapshot or mount the local webR library bundle before browser-side analysis.

Cache and reuse

Analysis cache

After browser DESeq2 or fgsea finishes, go to Methods & Export and select Export cache. The JSON cache restores sample metadata, DESeq2 contrasts, and fgsea result sets later without rerunning the browser analysis.

Loading a saved cache

Use Load cache in Methods & Export to reload a previous analysis cache. Cached sample metadata is restored when its sample_id values match the current count matrix.

Package library bundle

Runtime & Packages can mount the local webR library bundle for the configured package/library snapshot. This avoids reinstalling the DESeq2 and fgsea dependency closure during that browser session.

What is not cached

The analysis cache includes sample metadata when available, but it does not include uploaded count matrices, GMT files, or webR packages. Keep those input files separately, and mount or install packages again after a fresh browser session if needed.

Privacy and reuse

Uploaded count matrices, manifests, GMT files, mounted package libraries, and browser-generated results stay in the current browser session unless the user exports a CSV or analysis cache. The standalone report can be opened directly for static plots and tables; webR analysis is most reliable from an HTTP(S) page or a compatible local static server.
